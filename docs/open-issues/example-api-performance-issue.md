# API 性能问题 - 示例模板

## 问题描述

描述需要解决的具体性能问题或错误。包括症状、受影响的功能和用户影响。

示例:"API 端点 `/api/data/process` 在正常负载条件下间歇性出现高延迟(>5 秒),导致用户超时错误。"

## 根本原因

对导致问题的详细分析。包括:
- 技术根本原因
- 促成因素
- 触发问题的系统条件

示例:"需要对复杂连接进行数据库查询优化。当前查询在没有适当索引的情况下扫描整个表。"

## 影响评估

- **严重性**: 严重/高/中/低
- **受影响用户**: 受影响用户的百分比或数量
- **业务影响**: 收入、用户体验或运营影响
- **变通方法**: 当前已实施的任何临时解决方案

## 建议的解决方案

### 技术方法
详细的技术解决方案,包括:
- 所需的代码更改
- 架构修改
- 数据库模式更新
- 预期的性能改进

### 实施计划
1. **阶段 1**: 初步修复(时间线)
2. **阶段 2**: 优化改进(时间线)
3. **阶段 3**: 监控和验证(时间线)

## 测试策略

- **单元测试**: 验证修复的具体测试用例
- **集成测试**: 端到端测试场景
- **性能测试**: 负载测试和基准测试
- **回归测试**: 确保现有功能不会被破坏
